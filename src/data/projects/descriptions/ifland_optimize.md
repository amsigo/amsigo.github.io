# 프로젝트 소개
이 페이지는 ifLand의 플레이아이템과 관련한 최적화 내용을 담고 있습니다.

ifland의 플레이 아이템의 성능 개선을 위해   
CPU Usage, Draw Call, Memory 등을 프로파일링 하는 방법을 문서화 하고   
해당 내용을 팀원들에게 공유하고 팀원들을 리드하여 프로파일링을 진행 하였습니다.

프로파일링 결과들을 분석하여 성능 저하가 발생하는 부분의 원인을 분석하고   
성능 개선 방안을 제시하여 성능 개선 작업을 진행 하였습니다.

# CPU Usage 증가 관련 이슈 예시


### 아바타 코스튬 아이템을 장착 이슈

### 이슈
마네킹 아이템은 코스튬을 장착 해볼 수 있는 아이템으로 해당 아이템을   
Unity Profiler를 이용하여 성능을 측정 하였습니다.

![CPU_Profiler](./cpu_profiler.png)

### 원인   
코스튬 아이템을 반복해서 장착할때 성능 저하가 발생하여 해당 부분의 원인을 파악 하였고
코스튬을 장착할때 업데이트된 아바타의 전체 정보를 다시 갱신하고 있어   
 성능 저하가 발생 하는것을 확인 하였습니다.

### 해결
아바타의 코스튬 정보들을 캐싱하여 변경된 아바타 코스튬 정보만 업데이트 하는 방식으로   
개선하여 성능이 2배이상 증가 하였습니다.



# Draw Call 증가 관련 이슈 예시

### 점프맵 블럭 Draw Call 이슈

### 이슈
if Land의 점프맵 아이템은 블럭들을 여러개 적절한 위치에 배치하여 제작합니다.

점프맵의 DrawCall을 측정할때 DrawCall이 지나치게 높게 나와 Frame Debugger를 통해   
DrawCall 증가 원인을 분석 하였습니다.

![CPU_Profiler](./frame_debugger.png)


### 원인   
프로젝트의 Dynamic Batching 옵션을 사용하지 않아 동일한 Material을 쓰는 블럭이   
Batching이 되지 않아 DrawCall이 증가 하고 있었습니다.

### 해결
Dynamic Batching 옵션을 사용하지 않고 개선 하는 방안을 요청 받아   
Material의 Shader에 GPU Instancing을 추가하여 GPU Instancing 옵션을 활성화 시켜주었고

이로인해 점프맵 아이템의 DrawCall이 상당수 주는 효과를 볼 수 있었습니다.

# Memory 증가 관련 이슈 예시

### 이슈
if home에서 동일한 아이템을 배치 삭제 반복 테스트를 진행하며   
Memory Profiler를 통해 성능 측정을 하였고 반복할때마다 메모리가 증가하는 이슈가 있었습니다.

![Memory_Profiler](./memory_profiler.png)

### 원인
아이템을 배치하기 위해 위치를 지정하거나 삭제를 위해 수정모드로 들어갈때   
아이템이 Preview 모드가 되는데 이 과정에서 Preiview 모드로 바뀐 오브젝트는   
원래 배치된 아이템과 별개로 아이템이 copy 되어 Preview 모드용 아이템과 배치용 아이템으로   
나눠지는데 아이템을 삭제시에는 Preview 모드 아이템은 같이 삭제되지 않고 남고 있어   
메모리가 증가 하는것을 확인하였습니다.

### 해결
담당자에게 해당 내용을 전달하여 아이템 삭제시에 같이 남지 않도록 수정하여 해결 하였고   
메모리 증가 현상을 해결 하였습니다.
