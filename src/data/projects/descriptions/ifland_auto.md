# 프로젝트 소개
이 페이지는 ifLand에서 테스트 자동화를 구현한 내용을 담고 있습니다.

ifhome이라는 개별 공간 콘텐츠를 제작할때 성능적으로 문제 없는   
적절한 아이템 배치 수 조사에 관련된 테스트를 요청 받았습니다.

다양한 테스트 케이스를 직접 수동으로 하기에는 시간이 너무 많이 걸려   
해당 테스트 과정을 자동화 하였습니다.

## 사용 기술
- Appium
- Python
- RabbitMQ

## 프로젝트 구조
이 프로젝트는 크게 앱을 컨트롤하는 Client와  
어드민 수정이나 아바타 배치를 자동으로 서버 컴퓨터에서 처리하는 Server
두 가지로 나누어집니다.
![프로젝트 구조](./Project_Structure.png)

## 테스트 시나리오 동작 순서
1.앱을 실행합니다.   
2.어드민 페이지 실행 및 ifhome 홈 아이템 배치 변경   
3.아바타 매크로 실행 및 더미 아바타 입장   
4.if home 진입   
5.프로파일링 진행 및 데이터 기록   
6.테스트 종료 및 Slack 메시지 알림   
![프로젝트 플로우](./Project_Flow.png)

## 앱 자동화
앱의 자동화는 Appium을 이용하여 실제 디바이스를 테스트 케이스대로 버튼들의   
xPath를 이용하여 버튼을 찾고 클릭하도록 구현하여 ifhome에 진입하는 과정을 자동화 하였습니다.   
AOS기기 3개 IOS기기 3개를 연결하여 동시에 6개의 기기를 같은 시나리오로 테스트 하였습니다.

![동시 테스트](./Device_Test.png)

## 매크로 자동화
어드민 매크로와 더미 아바타를 배치하는 아바타 매크로는 서버 컴퓨터에서 처리합니다.   

Client를 실행하는 컴퓨터에서 RabbitMQ를 이용하여 서버 컴퓨터에 작업을 등록하고   
서버 컴퓨터에서는 등록된 매크로 작업을 순차적으로 실행합니다.

![RabbitMQ](./RabbitMQ.png)

## 프로파일링
앱에 포함시켜둔 프로파일러 기능을 활성화 시켜 프로파일링을 진행합니다.
![프로파일링](./Profiling.png)


## 테스트 종료
테스트의 진행 상황을 Slack을 통해 알림 받습니다.
![프로파일링](./Slack.png)

# 성과
수동으로는 어려울 수백가지의 테스트 케이스를 테스트 자동화를 구현하여   
2~3일동안 자동으로 테스트를 진행하여 ifhome의 적절한 아이템 배치수를 찾을 수 있었습니다.

그외에도 해당 자동 테스트를 응용하여 한개의 플레이아이템을 여러 유저가 동시에 사용시   
재현이 어려운 간혈적으로 발생하는 네트워크 동기화 이슈를   
자동 반복 테스트로 재현하여 해당 상황에 대한 이슈를 공유하여 해결 하였습니다.
